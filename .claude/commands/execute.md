Route [$ARGUMENTS] by tier — the argument's shape identifies the context (`work/README.md` § The command family). The meaning of this command at every tier: **advance the target through its workflow from wherever it stands** — the document is the state machine.

- **Milestone tier** — the argument references a `work/milestones/<n>.<name>/` directory (or "M<n>"): you are the [executor](.claude/agents/executor/CLAUDE.md) driving the [execute-milestone workflow](.claude/agents/executor/workflows/execute-milestone.md) — a Reconnaissance checkpoint that hardens the specification with the user, then one autonomous Manager child session per phase (`claude -p "/execute <phase-doc> automatically"`), looped with ultimate-acceptance reasoning until the milestone's acceptance criteria are satisfied. Preserve your context for objective-level oversight; delegate all building. Refuse an **open milestone** (`mode: open`) — there is nothing to converge on; its work rides individual work orders.

- **Work-order tier** — the argument references a `work/todos/….md` file: you are the [manager](.claude/agents/manager/CLAUDE.md) running the [work-order workflow](.claude/agents/manager/workflows/execute-work-order.md), type-polymorphic on the document's `type:` frontmatter (feature | bug | refactor | chore) and entry-state-routed on its accumulated sections (no plan → full sequence; approved plan → enter at implementation; `## Progress` ledger → resume where it stopped). Preserve your context window; dispatch specialists.

- **Single phase of a milestone** — the argument references a *phase doc* (`work/milestones/<n>.<name>/<p>.<phase>.md`): you are the [manager](.claude/agents/manager/CLAUDE.md). Seed a work order from it first, exactly as an Executor would (`work/todos/<date>.<phase-slug>.md`: acceptance criteria verbatim, `difficulty` carried, the MILESTONE.md `sessions:` list copied so the architect consult handle travels) — the phase doc stays the spec/record — then run the work-order workflow on the seeded order. This executes one phase without campaign ceremony; use `/execute <milestone-dir>` when the whole milestone should run.

  If — and only if — the arguments carry the literal `automatically` switch, apply the [work-order automatic-mode overrides](.claude/agents/manager/workflows/execute-work-order-automatically.md): the same sequence run without the sign-off pause, looping review→resolve→inspect to the order's target criteria and reporting a structured Acceptance Verdict. Absent the switch, run the manual workflow exactly as always.

If the tier is genuinely ambiguous, ask before proceeding — never guess a tier.
