# Footgun ledger

The active mission's footgun ledger: load-bearing constraints and traps discovered during work, one file per subsystem, indexed from the root `CLAUDE.md`. Planners and implementers **must** read the ledger file for any subsystem they touch.

Empty at present — this repo's agentic work begins with the agentic-workflow series; entries accumulate as work surfaces them. At mission close the ledger is archived to `work/missions/<v>/footguns/`; at the next mission's open, every entry is triaged: **carried** (still true — moves into the fresh ledger, re-certified), **designed-around** (the new design removes the trap; stays archived as the design's rationale), or **watch** (relevance unknown until the subsystem is touched). Protocol: the mission-tier proposal, Position 3.
