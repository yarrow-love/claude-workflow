# `.claude/` docs cite a root-`CLAUDE.md` footgun index the template no longer models

Filed 26-07-24 by the M3 phase-2 reviewer. Not investigated — symptom and locations only.

## Symptom

Eight product/scaffold files instruct roles to use, or reconcile, a **footgun index in the root `CLAUDE.md`**:

- `.claude/agents/manager/CLAUDE.md:156` and `.claude/agents/manager/workflows/execute-work-order.md:34` — "reconcile the footgun ledger (`work/footguns/` + the CLAUDE.md index)"
- `.claude/agents/executor/CLAUDE.md:90` and `.claude/agents/executor/workflows/execute-milestone.md:9,21` — same phrasing
- `.claude/agents/reviewer.md:18` — "`work/footguns/`, indexed from CLAUDE.md"
- `.claude/agents/planner.md:13`, `.claude/agents/implementer.md:13` — "indexes the footgun ledger in `work/footguns/`… the index table maps subsystems to files"
- `work/footguns/README.md:3` — "indexed from the root `CLAUDE.md`"

M3 phase 2 replaced this repo's root `CLAUDE.md` footgun-index table (D4 doctrine rewrite) with a `## Standing hazards` pointer at `.claude/ADR.md` § Standing hazards, plus a note that a mission's own ledger rides its branch. So in *this* repo the cited index table no longer exists.

## Why it is not simply a broken reference

These instructions remain **correct as generic convention**: a consuming repo authors its own root `CLAUDE.md` at bootstrap step 3 and may well index its `work/footguns/` ledgers there. `work/footguns/README.md` ships as a scaffold stub and reads correctly for that audience.

The drift is confined to the **template repo's own** root doc. The open question is a design one, which is why this is filed rather than fixed in phase 2 (it was outside that phase's declared scope, items 1–5 never naming the agent-role docs): **post-purification, should the template still model the ledger-index convention in its root `CLAUDE.md`, or should the generic instructions be rephrased so they do not presume a table that a purified template lacks?**

Either resolution is defensible; both touch agent policy prose across eight files, which wants its own blessed order rather than an end-of-phase edit.

## Where

- The eight paths enumerated above.
- The replacement text that created the drift: root `CLAUDE.md` § Standing hazards.

## Note for whoever fixes it

Phase 3's rehearsal will not surface this — that phase checks file presence and purification, not prose coherence. Do not assume it gets caught downstream.
