# Dispatch policy — the right agent, with the right context, for the task

## Refactor Objective

**What.** A deliberate, written dispatch policy for the Manager (and Executor): for each workflow step, which agent, **fresh or continued**, at which model tier. Token economy is a *consequence* of getting this right, not the frame — the frame is that fresh context and held context are different tools, and each step needs one or the other for a reason.

**Why.** Every fresh subagent pays full context-rebuild cost *and* loses the intent held by the agents who did the work; every continuation risks laundering away the independence that some steps exist to provide. The workflows currently choose per-step by convention inherited from the lasers era, not by stated principle. The principle, as now seeded in `manager/CLAUDE.md` § Delegation Guidelines:

- **Fresh when independence is the point** — reviewer, inspector. Never continued from a build context, regardless of token pressure.
- **Continued when the task extends held context** — documentation wrap-up, resolving review findings, plan questions (Planner Callback).

**Decided at reframing (26-07-11) — the documenter seed case.** Step 15 of `execute-work-order.md` no longer dispatches a fresh Documenter for wrap-up: the Manager **deputizes the implementer** (SendMessage, context intact) under the documenter's conventions, stamping `documenter@<machine>/<session>` (the role names the act). Fresh Documenter dispatches remain the norm for `/document` standing audits, where change-independence is a feature. Rationale: wrap-up transcribes *intent already in context* into docs; a fresh agent sees only the diff — the letter of the change, not the spirit.

**Questions this order still owns (at /plan):**

- Walk every step of `execute-work-order.md` and `execute-milestone.md` and classify: fresh-required / continuation-preferred / either. Write the classification into the workflow steps or the Manager doc (one place).
- The investigator on quality-gate failures (step 11): fresh, or continue the reviewer that surfaced the failure?
- Cost shape: measure per-specialist token spend from `tmp/accomplish/*.jsonl` terminal `result` events once real campaigns run in this repo — validate the policy against data, not intuition.
- Model pins review: planner→fable, implementer/designer→opus, rest→sonnet were lasers-era decisions; re-certify against this project's workload and the current model family.
- Interaction with per-section provenance: confirmed — a deputized agent stamps its **acting role** over the (unchanged, resumable) session id; already codified in `work/README.md` § grammar.

**Guardrail (standing, restated):** reuse must never launder independence away — reviewer and inspector stay fresh-dispatch; the rule is written into the Manager doc so token pressure can't erode it.

## Acceptance criteria (provisional — harden at /plan)

- A complete per-step dispatch classification in one authoritative place, with rationale.
- The step-11 investigator question decided and reflected in the workflow.
- Model pins re-certified or amended, with rationale recorded.
- Campaign cost data (once available) checked against the policy's predictions.

### Decision points

1. **Classification home** — chosen: one table in `manager/CLAUDE.md`; rejected: per-step inline annotations only (scatters the policy, invites drift). Consequence: workflows stay lean; one place to amend.
2. **Step 11 (gate failure)** — chosen: **manager judgment splits it** — an *obvious* failure (syntax error, missing import, path typo) goes straight to the continued implementer with the gate output attached (no investigator); a *non-obvious* failure gets a **fresh investigator** for diagnosis (failure diagnosis benefits from not inheriting the builder's assumptions — the same independence argument as review, applied to debugging), then the **continued implementer** executes the fix with the report. Rejected: always-fresh investigator (wasteful on trivial breaks); continue-the-reviewer (the reviewer's context is findings-shaped, not failure-shaped, and it burns review independence for later passes). Consequence: fastest path on trivial breaks, fresh eyes exactly where assumptions blind.
3. **Model pins unchanged** — chosen: re-certify fable/opus/sonnet as-is; rejected: adding a haiku tier for mechanical steps (premature — no cost data yet; revisit at the deferred validation). Consequence: zero churn now; the first campaign's cost events decide whether a cheaper tier is warranted.
