# Unification — one work-order workflow, polymorphic command routers

## Objective

Collapse the quadruplicated work-order workflows into one canonical sequence plus one overrides file, stand up the Architect as a first-class role directory, and replace the eleven-command family with the nine-command family behind the polymorphic `/plan` and `/execute` routers. This is the order with real design content: the merge must preserve the empirically proven step semantics *verbatim* — the value of the old workflows is that they are tried and true; the defect being fixed is only their multiplicity.

## Steps

1. **Write `.claude/agents/manager/workflows/work-order.md`** — the canonical manual sequence, merged from `build-feature.md` + `resolve-bug.md` + `refactor-codebase.md`, with `implement-todo.md` absorbed as an entry mode. Structure:
   - The shared sequence (read → investigate → format → interview → plan → **sign-off checkpoint** → implement → review → resolve findings per the consultation policy → quality gate → verify activation → inspect on `interface` scope → summary → document → milestone/footgun reconciliation → file to `_done/` → bug triage → commit discipline) — carried over word-for-word where steps are identical today.
   - A **per-type table** (`type: feature | bug | refactor | chore`) for the genuinely differing steps: investigation step (researcher vs investigator + report synthesis), document section names (Feature Request / Bug Report / Refactor Objective; Research vs Investigation Findings; Implementation vs Resolution vs Refactor Plan), close-out ritual (bugs: `resolution:` frontmatter + Resolution section; bug-closes-footgun rule). `chore` rides the feature shape minus ceremony.
   - **Entry-state routing** (absorbs `/implement`): a work order arriving with an approved plan section skips interview + planning and enters at implementation; the document state, not the command, decides. A `## Progress` ledger, if present, positions re-entry (crash-resumability).
2. **Write `.claude/agents/manager/workflows/work-order-automatically.md`** — the overrides, once (merged from the two `-automatically` files, which are near-identical today): escalate-on-novelty with the planner-consult ladder, the review→resolve→inspect convergence loop against acceptance/verification criteria (pass budget 3), quiet running, progress heartbeat, `## Acceptance Verdict` handback. Delete the six superseded workflow files.
3. **Stand up the Architect** — `.claude/agents/architect/CLAUDE.md` (role: design authority; mission + milestone tiers; interactive flagship sessions; the standing consult handle; never dispatches builds) + move `plan-milestone.md` into `architect/workflows/` + write `architect/workflows/plan-mission.md` (the charter protocol from the mission-tier proposal Position 4: objective/why-now, scope + non-goals, settlement reference, success **and failure** criteria → `work/mission.md`). Update `planner.md`'s Callback Chain section: the milestone-tier planning role is the Architect; stamps are `architect@… # milestone architect` / `# mission architect` (the grammar's role segment now matches the role ladder).
4. **Rename the Executor workflow** — `accomplish-goal.md` → `executor/workflows/execute-milestone.md`; dispatch string becomes `claude -p "/execute <phase-work-doc> automatically" …`; all internal references updated. Semantics unchanged — Reconnaissance gate, outer loop, ultimate acceptance, budgets all stand.
5. **Write the command routers** (delete `build`, `resolve`, `refactor`, `implement`, `accomplish`, `answer`):
   - **`plan.md`** — three-tier router table: mission (new mission / `work/mission.md` → Architect, plan-mission), milestone (`work/milestones/<n>.<name>/` or "M12"-style reference → Architect, plan-milestone), work order (`work/todos/….md` → Manager dispatches Planner, plan-todo semantics). Routing key: argument shape; when ambiguous, ask.
   - **`execute.md`** — two-tier router: milestone directory → Executor (execute-milestone); work-order file → Manager (work-order, manual; `automatically` literal switch → the overrides file). State the framing: *advance this document through its workflow from wherever it stands*.
   - **`investigate.md`** — Investigator dispatch for codebase questions and bug traces (absorbs `/answer`'s codebase half; `/research` owns the external half); answers inline or appends `## Answer` to a referenced file.
   - **`close.md`** — mission close per the mission-tier proposal Positions 4–7 **plus determination 7's docs/research triage**: closing report drafted with the operator → footgun/todo/proposal/docs triage → `git mv` into `work/missions/<v>/` → ledger row → tag. Hard rule stated in the command: operator-invoked only; the system never proposes or initiates it. (First invocation is far off; the protocol lands now so the tier is complete.)
   - `triage.md`, `document.md`, `research.md` — repoint and re-voice ("Manager"), semantics unchanged. `/archive` + `/initialize` land in order 5.
6. **Reconcile `plan-todo.md` and `triage-todos.md`** to the merged reality (workflow names, entry states, `chore` in the type enum — vocabulary already sanctioned in `work/todos/README.md`, order 1).

## Decision points

Settled by the determinations record (order 1) — the routers and roster implement decisions already signed off. One residual for execution time: whether `/execute` on a work order with **no plan** runs the full manual sequence (interview included) or refuses and points at `/plan` — lean: run the full sequence; that *is* the manual workflow's shape.

## Acceptance

- `.claude/commands/` contains exactly: `plan`, `execute`, `triage`, `investigate`, `research`, `document`, `close` (+ `archive`, `initialize` after order 5); no retired command names anywhere in `.claude/`.
- The manager workflow directory contains `work-order.md`, `work-order-automatically.md`, `plan-todo.md`, `triage-todos.md`, `audit-documentation.md` only; no step text duplicated across workflow files (spot-grep distinctive sentences).
- A written walkthrough in each router demonstrates correct routing for: `/plan M12`, `/plan work/todos/x.md`, `/plan` new mission, `/execute work/milestones/12.foo/`, `/execute work/todos/x.md`, `/execute work/todos/x.md automatically`.
- Executor dispatch string updated; a dry-read of `execute-milestone.md` reaches the child-session dispatch with no dangling references.
- Every step present in the old build/resolve/refactor/implement workflows is traceable into the merged files or explicitly noted as retired (diff-audit in the Summary — this is the "differences very clear" guarantee).
