# Headless automatic-mode Managers exit early — no `## Acceptance Verdict` on clean exit

## Bug Report

**Source**: second cross-repo report from the Orchestra executor (downstream bootstrap consumer),
relayed 26-07-23 by the operator through the `architect@play/76ba13c6…` consult handle. Dogfooding
evidence from a real 6-phase milestone under `/execute <milestone>`.

**Symptoms**: autonomous Manager child sessions (`claude -p "/execute <work-doc> automatically"
--output-format stream-json`) **terminate early mid-workflow, before writing `## Acceptance
Verdict`**, on phases involving real processes, wall-clock measurements, or subagent-heavy
convergence — while completing pure authoring phases in one pass. Observed: two authoring phases,
1 pass each; two build/measurement phases needed **4 and 3 passes**, every extra pass a distinct
mechanical cause, none a defect in the actual work. Impact: 3–4× token cost on affected phases,
and the autonomy promise inverted — the Executor babysits, tails, diagnoses, re-dispatches. The
**mtime-stall rule cannot catch this class**: the child exits *cleanly*, success `result` event,
no verdict.

**Four observed mechanical causes** (reporter-diagnosed; each recoverable only by an
Executor-noticed `--resume`):

1. **`&`-wrapped dispatch detaches stdio** — wrapping `claude -p …` in `… &` inside a backgrounded
   shell (instead of making the invocation itself the background task) detaches stdio; the child
   stops after ~5 turns.
2. **Background-and-wait ends the session** — a Manager that backgrounds a long measurement
   (RSS-leak loop, ≥15-min SSE hold, soak) and "waits" ends its turn with pending external work;
   the session terminates; the measurement completes ownerless.
3. **`pkill -f <path>` self-kill** — cleanup via `pkill -f 'src/…/foo.ts'` matches the session's
   own command line (the `-p` prompt embeds the path); exit 144. Not headless-specific — any
   session whose prompt embeds a path is exposed.
4. **Turn-end after subagent return (deepest)** — even with 1–3 fenced, a Manager sometimes emits
   a status message and ends its `-p` turn after a specialist returns, before the remaining
   convergence loop (review→gate→verify→verdict); observed `num_turns` 5 and 11 with real subagent
   work done and no verdict. Reliable recovery: an Executor-authored `--resume` mandating direct
   tool-call completion (no further subagent dispatch), "do not stop until the verdict is written."

**Reporter's suggested changes** (accepted in substance by the consulted architect; shaping notes
below): an **EXECUTION DISCIPLINE** subsection in `execute-work-order-automatically.md`
(foreground-block under the Bash ceiling vs detach via `setsid` with PID+log captured and
Executor-staged finalization — never background-and-wait; no `pkill`/`killall -f` on name/path
substrings — exact captured PIDs or port-derived PIDs via `ss -tlnp` only; the dispatch invocation
*is* the background task, never `&`-wrapped); **premature-exit detection** in
`execute-milestone.md` beside mtime-stall (on a child's terminal `result`, check the work doc for
`## Acceptance Verdict`; absence on clean exit = early stop → `--resume` to continue, distinct
from re-plan); a bounded **resume-until-verdict** loop as standard child handling; a canon note
that measurement/subagent-heavy phases may need a final-mile `--resume` mandating direct
in-session work.

## Filer's assessment (consulted architect)

- **Cause 4 survives mission v1's conversion** — a subagent Manager can equally stop short of its
  verdict; post-conversion the remedy becomes a SendMessage nudge to a paused-in-place Manager
  instead of a `--resume`. Verdict-absence detection is therefore **permanent canon**,
  dispatch-mechanism-independent — write it once, mechanism-neutral. Causes 1–2 are
  headless-specific (post-M2 they survive only in the documented-fallback path); cause 3 is
  universal and belongs where every role inherits it.
- **The final-mile direct-work instruction** must be worded as a bounded, resume-scoped exception
  to the Manager's "don't implement directly" constraint (mechanical convergence-mile work only) —
  not general license.
- **Operator decisions flagged** (for adoption, per the consult protocol): routing/timing (land the
  discipline + detection rules on `main` promptly vs fold into the M2 `execute-milestone` rewrite);
  the resume-until-verdict bound's relation to the outer pass budget (shared counter vs its own);
  and the final-mile exception's wording latitude (narrowest = direct tool calls for mechanical
  remaining steps only, under explicit Executor instruction, never self-invoked).
