**Caveat (honesty):** default/ceiling numbers come from headless `claude -p` subprocesses; the env-var *mechanism* is confirmed functional there (var=1 caps, var=2 permits, no-var deep). The **interactive** default (plain `claude`, no var) was **not** directly certified — only the interactive *enabling* direction (this session, var=2) was. Whether interactive plain-session default matches the `-p` default-on observation is an open item M2 should confirm before relying on it.

**Why adapted-with-note, not fail:** a probe-1 *fail* = the load-bearing capability (nested depth-2 dispatch) does not work → mission failure-criteria path.
