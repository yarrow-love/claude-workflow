# Audit `.claude`, unify agent terminology, and apply the mission-tier work system

## Refactor Objective

**What.** Audit the entire `.claude/` directory, reconcile the agentic system's terminology
into one coherent scheme, and apply the `work/proposals/26-07-05.agentic-workflow` proposal
(anchored by `26-07-02.mission-tier.md`) so the documented work system and the on-disk reality
agree. This is the keystone of the slate-clearing effort: it makes the `work/` machinery that
the other two orders are filed under *actually official*.

**Why.** The `.claude` system was authored for the previous project (astrology.observer) against
a **flat layout** (`todos/`, `docs/milestones/`, `docs/footguns/`). The reconstruction already
moved the work system toward the mission-tier layout (`work/{proposals,todos,milestones,missions}/`
exist), but nothing in `.claude` was updated to match, and several references are now dangling.
The agentic vocabulary is also split across two registries that must be unified.

---

### Audit findings (initial, from filing-time survey)

**A. Dangling / broken references — the migration is half-done:**
- The mission-tier layout dirs exist under `work/` (`proposals/`, `todos/`, `milestones/`,
  `missions/`, `notes/`), but the old flat dirs (`todos/`, `docs/milestones/`, `docs/footguns/`)
  **do not exist** — yet **15 files in `.claude/` still reference the old paths**
  (`grep -rl 'todos/\|docs/milestones\|docs/footguns' .claude/`): commands (`plan`, `triage`,
  `answer`) and most orchestrator/executor workflows + agent docs.
- **`.claude/hooks/` does not exist**, but `settings.json`'s `SessionStart` hook runs
  `bun run "$CLAUDE_PROJECT_DIR/.claude/hooks/git-sync-check.ts"` — **the hook is broken today.**
  (Also depends on the repo existing at all — see `26-07-11.relink-git-remote`.)
- **`.claude/agents/PROJECT.md` does not exist**, but `orchestrator/CLAUDE.md` instructs every
  role to read it at session start (quality gate, front-end surface, runtime-evidence locations,
  per-role notes). Must be authored for this project.
- **`todos/README.md` (now `work/todos/README.md`) does not exist**, but it is the single most
  referenced doc in the system — the filing convention, disposition/priority vocabulary, and
  document-structure spec all live there per the workflow docs. Must be authored (it also
  retroactively formalizes the two sibling orders' frontmatter).
- **No root `CLAUDE.md`** — the orchestrator doc calls it "the architecture source of truth
  (auto-loaded)." Absent. Needs authoring (can be deferred to the re-architecture phase, but the
  reference should at least be reconciled).
- **`BACKLOG.md`** (the triage ledger) does not exist yet — expected, created on first `/triage`.

**B. Terminology to unify** — the "adjust role names, architect a unified agentic system" ask:
- **Two agent registries to reconcile:** the harness agent registry (this environment exposes
  `orchestrator`, `executor`, `scheduler`, `planner`, `researcher`, `investigator`, `reviewer`,
  `designer`, `inspector`, `implementer`, `documenter`, `resolver`, plus `Explore`/`Plan`/
  `general-purpose`) vs. the project's `.claude/agents/*` definitions. Names mostly align — the
  task is to make them *deliberately* one system, not coincidentally overlapping, and to decide
  the canonical set (e.g. is `Explore`/`Plan` used, or only the custom roles?).
- **The tier ladder** from the mission-tier proposal (`work order → milestone phase → milestone
  → mission`) with its command family (`/build` inner loop, `/accomplish` outer loop,
  `/plan <milestone>`, `/complete <mission>`, `/triage`, `/resolve`, `/schedule`) — verify the
  commands present (`accomplish, answer, build, document, implement, plan, refactor, research,
  resolve, schedule, triage`) map cleanly onto the ladder, and add what the proposal implies but
  is missing (notably a charter/`/complete` mission-close command, per Positions 4 & 6).
- **Proposal-introduced roles** referenced in the corpus but absent from `.claude/agents/`
  (e.g. `workflow-designer` appears in proposal frontmatter `agent:` callbacks) — decide whether
  they become real roles or were session-scoped.
- Settle **mission/milestone naming** (proposal Open Question 3: `v<version>.<slug>`) and the
  **type vocabulary** for work orders — the current planning workflow only branches on
  `feature|bug|refactor`, but the two sibling orders needed `chore`; formalize the enum.

**C. Companion proposals to fold in** (each may spawn or inform sub-work — read before planning):
`26-07-03.accomplish-live-feed` (adopted; residual tee-through dispatch), `agent-callback-identity`,
`cross-lan-agent-calls`, `planner-callback`, `26-07-05.per-section-provenance`,
`26-07-06.durable-resumability`.

### Scope

- **In:** everything under `.claude/` (agents, commands, workflows, settings, hooks); authoring
  the missing keystone docs (`work/todos/README.md`, `.claude/agents/PROJECT.md`, hooks the
  settings reference); the path-reference sweep (old flat → `work/`); the terminology
  reconciliation; adopting the mission-tier layout conventions.
- **Out (defer):** the substantive re-architecture of the astrology system itself; writing the
  first `work/mission.md` charter (that is the *next* phase this clears the ground for); deep
  content rewrites of research docs.

### Considerations / open questions (for `/plan` — do not resolve at filing)

- This order is **partly self-referential**: it defines the work-system machinery it is itself
  filed into. Applying it retroactively legitimizes all three `26-07-11` filings. The
  proposal's "reference sweep is the real cost of the rename" (mission-tier §"What happens now")
  is the bulk of the mechanical work.
- Sequencing vs. `26-07-11.relink-git-remote`: the git repo should exist first so the migration
  commits land in version control and the `git-sync-check` hook has something to check. Treat
  relink as a soft prerequisite.
- Decide the fate of the flat-path references: rewrite in place to `work/…`, or is any workflow
  genuinely obsolete under the new tier model and should be retired rather than repointed?
- Whether to author the root `CLAUDE.md` now (stub) or at re-architecture time.

### Acceptance criteria

- Every `.claude/` reference to a path resolves to a real location under the mission-tier layout
  (no dangling `todos/`, `docs/milestones/`, `docs/footguns/`, `.claude/hooks/…`,
  `.claude/agents/PROJECT.md`, `work/todos/README.md` references).
- `settings.json`'s `SessionStart` hook runs cleanly (hook file exists and works, or the hook is
  removed/replaced with rationale).
- `work/todos/README.md` exists and documents the filing convention, frontmatter vocabulary
  (`status`/`type`/`difficulty`/`priority`/`disposition`/`aligns`), and document structure — and
  the three `26-07-11` orders conform to it.
- `.claude/agents/PROJECT.md` exists with this project's operational facts (quality gate,
  front-end surface, runtime-evidence locations, per-role notes).
- The agent roster and command family are documented as **one** coherent system: a single
  canonical role set, the tier ladder → command mapping complete (including mission-close), the
  work-order `type` enum formalized, naming conventions (mission/milestone slugs) settled.
- A short migration note / ADR records what moved, what was retired, and the terminology
  decisions — so the re-architecture phase starts from a clean, documented baseline.
