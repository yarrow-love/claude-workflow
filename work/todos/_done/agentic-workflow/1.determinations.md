## The determinations (locked 26-07-11)

| # | Determination |
| --- | --- |
| 1 | **Callback grammar**: `<role>@<machine>/<session-id>[#<message-id>]` — the session-id is the resume handle (`claude -p --resume` forks at full context); the `#message-id` fragment is a *citation* into the transcript (greppable in the archived jsonl), not a resume target. Field name: `sessions:` (append-only frontmatter list). |
| 2 | **Roles** (corporate ladder, all agentive nouns): **Operator** (owner; all ultimate judgment) → **Architect** (design authority, staff role — plans missions and milestones interactively at flagship, never dispatches builds; tier-qualified in stamp comments: `# mission architect`, `# milestone architect`) → **Executor** (executive — drives one milestone campaign, one Manager child-session per phase, reasons about ultimate acceptance) → **Manager** (line management, né Orchestrator — drives one work order to its Acceptance Verdict via specialists) → **specialists**: researcher, investigator, planner, designer, implementer, reviewer, inspector, documenter, **integrator** (né resolver — renamed to break the collision with bug-resolution). The **Planner** stays distinct from the Architect: same competency, different tier — the Planner authors the implementation-plan section of one work order as a dispatched subagent; the Architect authors specs (charters, milestone indexes, phase docs) as an interactive session. **Scheduler is retired** (its design contradicts the executor's session-nesting lesson; it is the seed of a future work-queue tier — deferred proposal). |
| 3 | **Commands** (nine): `/plan` (three-tier polymorphic: mission / milestone / work order), `/execute` (two-tier polymorphic: milestone → Executor, work order → Manager; absorbs build, resolve, refactor, implement, accomplish; `automatically` literal switch retained), `/triage`, `/investigate` (replaces `/answer`), `/research`, `/document`, `/close` (mission close — outcome-neutral, **operator-invoked only, never system-initiated**), `/archive`, `/initialize` (read-only environment audit). Retired: `/build`, `/resolve`, `/refactor`, `/implement`, `/accomplish`, `/answer`, `/schedule`. Convention: commands are verbs, agents are nouns; `/execute` means "advance this document through its workflow from wherever it stands" — the work order is the state machine. |
| 4 | **The portability seam**: `.claude/` holds *all* harness definitions (the harness only discovers agents/commands there) with `.claude/BASELINE.md` as the manifest of the generic, portable bootstrap — anything not listed is project-specific by definition. `work/PROJECT.md` is the project supplement (quality gate, front-end surface, runtime evidence, per-role notes). Root `CLAUDE.md` is the architecture source of truth. |
| 5 | **MCPs**: Playwright + Chrome DevTools configured at **user scope** (always-useful, like Context7); `/initialize` verifies presence; `work/PROJECT.md` carries only project specifics (URLs, test project, auth bootstrap). |
| 6 | **Durability**: transcripts are the artifact behind the stamps — archive-on-stamp to in-repo `work/sessions/<machine>/<uuid>.jsonl.gz` (+ sha256 manifest); `/archive` is the checkpoint sweep + backfill; optional SessionEnd hook automates; `git mv` into `work/missions/<v>/sessions/` at mission close. Store fork resolved *in-repo* for this single-machine repo (the central-store lean in the imported proposal was a lasers-studio answer). |
| 7 | **Mission close (`/close`)** gains **docs & research triage** alongside footguns/todos/proposals: `docs/` is *provisionally* eternal — each doc carried (stays) or archived (`work/missions/<v>/docs/`) at close, keeping the working tree lean because agents grep the working tree. |
| 8 | **`work/` layout**: `todos/` (kept, migrated from flat `todos/`), `proposals/`, `milestones/`, `missions/`, `runbooks/` (mission-scoped), `sessions/` (the archive), `notes/` (pre-proposal context: ideas not mature enough to propose, observations not settled enough to document — that one line is its charter), `PROJECT.md`, `README.md`, and eventually `mission.md`. **`objectives/` retires** (empty; real objectives live in `todos/` and `milestones/`; proto-ideas in `notes/`/`proposals/`). |
