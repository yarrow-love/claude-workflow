# Consults mutate the transcripts they address — add `--fork-session`

## Bug Report

**Symptoms**: every consult recipe in live canon runs `claude -p --resume <session-id> …` on the assumption that resume forks. Per current CLI docs (verified in the halted lan-intercom order's planning research, 26-07-22, `manager@play/34b58558-60d4-4d45-921f-3fe9f1a1ed88`): `-p --resume` **continues the transcript in place**; `--fork-session` is the flag that forks. Every consult therefore mutates the transcript it addresses — and archived transcripts are checksummed snapshots (`work/sessions/MANIFEST`), so a mutating consult silently diverges the live transcript from its archived copy and breaks the restore-before-resume ladder. Extracted from the halted `26-07-22.lan-intercom.md` so archive integrity doesn't wait on Orchestra.

**Affected surfaces** (from that plan's recipe sweep — re-verify by grepping `-p --resume` / `--resume` across `.claude/` and `work/README.md`):
- `.claude/bin/consult` (~line 79) — the `spawnSync` claude invocation.
- `work/README.md:85` — the callback-grammar doctrine claim ("`claude -p --resume <session-id>` **forks** the session") is false as stated; correct it to name `--fork-session` as what makes the fork true.
- `.claude/agents/manager/CLAUDE.md` (Planner Callback, ~line 123) — consult recipe.
- `.claude/agents/manager/workflows/execute-work-order-automatically.md` (~line 14) — escalate-on-novelty consult recipe.
- `.claude/agents/executor/workflows/execute-milestone.md` (~line 19, step 7.3) — architect-consult recipe.

**Constraint — do not blanket-add the flag**: the Executor's *residual re-dispatch* (`--resume <session_id>` of a crashed/incomplete Manager child, `executor/CLAUDE.md` + `execute-milestone.md` 7.3 PARTIAL branch) intends **continuation**, not forking — the child should pick up its own transcript. Leave those invocations unflagged, and add one sentence at each distinguishing continuation-resume (re-dispatch: mutation intended) from consult-resume (fork mandatory), noting that a continued child's archive entry refreshes at its next stamp/`/archive` sweep.
