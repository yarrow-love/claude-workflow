# Phase 1 — mechanics

## Objective

Land the mechanical substrate the canon rewrite will cite as fact: the D2-re-scoped depth-cap env key (settings + BASELINE + `/initialize` audit) and the `session-archive` subagent-transcript sweep, self-tested. Close M1's interactive-default caveat via the operator-gated check. No canon prose moves in this phase — `executor/`, `manager/`, and `commands/` docs are untouched.

## Scope

**1. The depth-cap key.** `.claude/settings.json` gains a new top-level block beside `permissions` and `hooks` (none exists today):

```json
"env": { "CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH": "2" }
```

Rationale (this is the D2 re-scope, operator-confirmed 26-07-24, and the BASELINE text must say it plainly): the key is a **pin + cap**, not an enabler — M1 certified that on v2.1.218 nesting is default-ON and deep (≥3 levels) and the var restricts. Value `"2"` encodes the ladder exactly (root session → Manager → specialists); it pins behavior against platform default flips (twice in one year) and is the blast-radius principle applied to dispatch. Note the structural mechanism M1 observed: at the cap, a subagent is simply not granted the Agent tool — no error is emitted.

**2. BASELINE + `/initialize`.** Append one sentence to `BASELINE.md`'s settings-skeleton clause (house style: clause-per-baseline-item prose, placed with the baseline keys, **not** in the "Project decisions, not baseline" carve-out) naming the key, the value, and the cap rationale. Add one `/initialize` check in `commands/initialize.md` matching the house pattern (state the target; name the fix/report on failure): the key present in `.claude/settings.json` and set to `"2"` (missing/other → report with BASELINE's recipe). The audit is a value-pin check — do not write a behavioral "is nesting on?" probe; that M1-era framing is obsolete.

**3. The subagent sweep.** Extend `.claude/bin/session-archive` (83 lines, single `archive()` primitive called from two sites — hook mode and CLI-arg mode; both must gain the sweep): after archiving the main transcript for `<uuid>`, glob the parent-scoped subagent files at `~/.claude/projects/<slug>/<uuid>/subagents/agent-*.jsonl` and `agent-*.meta.json` (M1-confirmed: depth-2 transcripts store **flat** in this same directory, keyed by the root session — one glob covers all depths; the `.meta.json` sidecars carry `agentType`, `description`, `spawnDepth` — exactly what an Orchestra peer reading an archived tree needs, so they are swept too). Archive each gzip'd to `work/sessions/<machine>/<uuid>/subagents/`, mirroring the live layout, one MANIFEST line each (the existing upsert-by-path + sort semantics extend unchanged). Explicitly exclude the sibling `tool-results/` directory. A session with no `subagents/` dir archives exactly as today — the sweep is additive, never a failure path. Footgun (ledger: [`work/footguns/bin-scripts.md`](../../footguns/bin-scripts.md)): no `*/`-containing token in the JSDoc header — describe globs in prose there.

**4. Sweep self-test (in-session, headlessly verifiable).** The executing Manager session necessarily dispatches specialists, so its own `subagents/` dir is a live fixture: after implementing, run `bun .claude/bin/session-archive <own-session-id>` and verify (a) `work/sessions/play/<own-session-id>/subagents/` contains `agent-*.jsonl.gz` (+ meta) for the specialists dispatched this session, (b) `cd work/sessions && sha256sum -c MANIFEST` passes, (c) the no-arg usage/exit-1 contract still holds (quality-gate smoke test).

**5. Interactive-default check *(operator-gated)*.** M1 certified default-ON only via `-p` subprocesses; the interactive no-var default is uncertified. Staged recipe for the operator (must run in a **no-var environment** — i.e. any directory whose settings do not set the key, e.g. `$HOME`; once this phase lands the key, sessions in *this* repo always run capped): launch plain interactive `claude`, dispatch a subagent that dispatches its own subagent, record whether depth-2 completes. Record the observation verbatim in the phase work order. This is evidence, not a gate — either outcome closes the caveat; a divergence from default-ON only tightens the BASELINE rationale wording, it does not block the phase.
