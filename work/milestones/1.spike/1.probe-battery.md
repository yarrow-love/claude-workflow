# Phase 1 — probe-battery

## Objective

Run the five empirical probes of subagent dispatch on the pinned CLI and record a reproducible verdict for each in the work order. The phase produces evidence, not changes: nothing in canon, settings, or tooling moves.

## Scope

**Choreography.** The whole phase runs **operator-present**: the probe parent is an interactive session the operator launches with the env var set, and the phase work order is executed inside it. The Executor stages this recipe at the campaign's reconnaissance gate rather than dispatching a headless Manager:

```sh
CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH=2 claude   # then: /execute work/todos/<date>.<phase-slug>.md
```

That Manager session is simultaneously the probe parent — it dispatches the nested tree, sends the mid-turn nudge, is watched in the panel, and is the session ended and resumed for the durability probe. Probe subagents do harmless, observable work only (read files, report identity); they never mutate the repo.

**Step 0 — environment pin (operator-gated).** Operator upgrades the CLI to latest; `claude --version` output recorded in the work order before any probe runs. Precondition for everything below: version ≥v2.1.217.

**Probe order** — fail-fast on the load-bearing, panel attestation continuous, session-ending probe last: **1 → 5 → 2 → 3 (attested across 1/2/5) → 4**.

1. **Depth** (gates the battery): *control arm* — in a session launched *without* the env var, a dispatched subagent attempts to dispatch its own subagent; expected: unavailable/refused (nesting off by default — the check M2's `/initialize` audit inherits). *Gated arm* — in the env-var probe parent, a depth-1 subagent successfully dispatches a depth-2 subagent that completes a verifiable task. Record CLI version + exact behavior of both arms. **Fail here → stop the battery, escalate per the charter's failure criteria.**
2. **Mid-turn nudge**: with a deliberately long-running depth-1 subagent active, SendMessage it mid-run; record delivery timing and interjection semantics (documented expectation: the launcher's message is treated as task direction). Repeat toward a completed subagent (documented: auto-resumes) for contrast.
3. **Panel depth** *(operator-gated)*: the operator attests, in the work order, what was live-visible in the panel during probes 1/2/5 — depth-1 and depth-2 transcripts, task status, permission surfacing. This probe is observation, not a separate dispatch.
4. **Durability** (last — it ends the parent): record the `~/.claude/projects/<slug>/<sessionId>/subagents/agent-*.jsonl` paths for the probe tree; write all verdicts gathered so far to the work order (crash-safe); end the parent session; `claude --resume <parent>` — deliberately *without* `--fork-session`, continuation is the probe — then continue a named subagent by id via SendMessage and verify its context is intact (it must reference earlier in-run facts). Record the probe 4 verdict from the resumed session. **Fail here → escalate per the charter's failure criteria.**
5. **Model + permission**: depth-1 dispatched with one model, depth-2 with another; each reports the model it runs on (system-prompt self-identification), checked against the per-call overrides. Permission: verify inheritance matches documented dominance — the parent's mode (this repo: `bypassPermissions`) overrides any stricter subagent-level setting; record what was attempted and observed.

**Verdict write-up.** Each probe closes as pass / fail / adapted-with-note with its recipe inline (commands, dispatch prompts, observed outputs) so a future session can reproduce it verbatim. The work order ends with the premise-reconciliation table (proposal § The verified premise 1–5 → confirmed / diverged on the pinned version).

## Acceptance

- Work order records the pinned CLI version (`claude --version` output), ≥v2.1.217, dated before the first probe.
- Probe 1: both arms recorded — control (no var: nesting off) and gated (var=2: depth-2 dispatch completed) — with exact observed behavior.
- Probe 2: delivery semantics recorded for a running and a completed subagent, against the documented expectations.
- Probe 3 *(operator-gated)*: operator's attestation of live panel visibility at both depths, in the work order.
- Probe 4: subagent transcript paths recorded; parent ended and resumed; a subagent continued by id with demonstrated context retention.
- Probe 5: per-call model honored at both depths (evidence quoted); permission-inheritance observation recorded against the documented dominance rule.
- Every verdict is pass / fail / adapted-with-note; adapted-with-note names the proposal premise it touches; any fail on probes 1 or 4 is recorded as an escalation to the mission's failure-criteria path — the phase must not close around it with a workaround.
- Premise-reconciliation table present (5 rows, confirmed / diverged).
- No repo changes beyond the work order and this milestone's docs: no settings, canon, or BASELINE edits (verifiable via `git status` / diff scope).
