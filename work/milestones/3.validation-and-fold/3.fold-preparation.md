# Phase 3 — fold-preparation

## Objective

Stage the fold so `/close` runs as pure ceremony: a file-level purification manifest, a fold recipe **rehearsed to green in a disposable worktree**, and a consolidated close runbook with prepared triage dispositions. M3 stages and rehearses; the operator performs. This phase also carries the campaign's Executor-held judgment item — the real upward pause-and-ask that completes criterion 3.

## Scope

**1. The purification manifest** — a file-enumerated keep/strip ledger for the `main`-bound tree, grounded in the planning inventory (26-07-24) and re-verified against the live tree at execution:

- **Keeps (product)**: `.claude/` whole (incl. the new `ADR.md`, `bin/commit-guard`, expanded `consult`); rewritten root `CLAUDE.md` + `README.md`; `.gitignore` (with the secret patterns from `970ff22`), `.cbmignore`; `src/README.md`; `docs/README.md` + `docs/research/README.md`; `work/` **scaffold exactly per BASELINE's reconciled step-2 enumeration** — `work/README.md` (machine registry reset to a placeholder row per BASELINE's stated adaptation), the canon directory READMEs (`todos/`, `proposals/`, `milestones/`, `missions/`, `design/` + `system/` + `sessions/`, `footguns/`, `notes/`, `runbooks/`, `sessions/`), empty `_done/`/`_cancelled/`/`bugs/`/`_deferred/`/`_rejected/`/`_enacted/` containers.
- **Strips (instance — including `main`'s own residue; main is not pure today)**: `work/MISSION.md`, `work/PROJECT.md`, all dated todos + `_done/` content (founding series included), all dated proposals + `_enacted/26-07-05.agentic-workflow/` + `_deferred/` content, `work/milestones/*` (all three), `work/missions/v1…/` (the close ceremony's own archive — it lives on the branch), `work/footguns/{bin-scripts,consult}.md` (absorbed by ADR § Standing hazards), `work/runbooks/{probe-battery,<close-runbook>}.md`, `work/sessions/MANIFEST` + `play/`, `work/design/` session content beyond scaffold, `work/notes/` content beyond its README.
- **Inventory item**: the tracked top-level `.archive` path — inspect, classify product/instance, disposition it in the manifest (unexamined at planning).
- **Executor-held item (deliberate — the criterion-3 upward leg)**: the fold's **presentation on `main`** — true merge (`git merge 26-07-23` + purification commit; honest shared ancestry, the mission's blobs in `main`'s history) versus squash-fold (one synthetic commit; slim history, no shared ancestry with mission branches). The phase Manager must **not** decide this: it pauses-and-asks the Executor mid-campaign (ending its turn with the trade-off stated), the Executor answers by SendMessage (consulting the architect handle if wanted), and the exchange is recorded in the criterion-3 ledger. *(The planning session's lean, for the Executor's deliberation, not binding: true merge — ancestry is worth more than slimness in a docs-scale repo.)*

**2. The fold recipe** — exact command series, written to be executed by the operator (or their session) after `/close`: fetch-fresh state assertions (`git status` clean, tag present), the merge (sole known conflict surface: `b332083` touched `.claude/` on `main` — pre-resolved instruction), the purification commit (enumerated `git rm` paths from the manifest — never `git add -A`), the template quality gate on the result (reference-resolution grep against the bare tree with the reconciled sanctioned-absence list; `bun` tooling smoke tests; `/initialize` audit), and the push. Every step idempotent or safely re-runnable; every failure mode named with its recovery.

**3. The rehearsal (the acceptance instrument).** In a disposable worktree (`git worktree add <scratch> main`), execute the recipe end-to-end against a throwaway branch: merge, purify, run the full template gate there. Record the gate output in the work order; remove the worktree. The rehearsal must reproduce a `work/` tree that matches the reconciled BASELINE enumeration **exactly** (`diff` the file listing against the manifest's keep column). A red rehearsal blocks the phase — fix the manifest/recipe, not the gate.

**4. The close runbook** — `work/runbooks/fold-and-close.md`, consolidating for the operator: pre-flight (`/archive` sweep green, phase 1's spot-check re-run); the `/close` ceremony as written (closing report drafted in dialog; triage with **prepared dispositions**: `26-07-11.configure-command` → archive open-carried, re-seed on a future mission branch; `26-07-22.lan-intercom` → archives halted, Orchestra valve untouched; footguns → absorbed-by-ADR noted; runbooks → retire with mission; docs/research → nothing to triage); the archive `git mv` into `work/missions/v1.subagent-boundary/`; the ledger row; the tag `v1.subagent-boundary-closed`; then the fold recipe; then post-fold verification on `main`. The runbook is instance content — it archives with the mission it closes.
